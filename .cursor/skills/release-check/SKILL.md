# Release Check

## 목적

배포 직전 점검에서 누락과 리스크를 줄이기 위한 최종 검증 절차를 제공한다.

## 사용 시점

- 실제 배포 직전 최종 확인 단계
- 스테이징 환경 점검
- 릴리즈 후보(Release Candidate) 검토

## 실행 원칙

1. 기존 프로젝트 구조, 스택, 네이밍을 우선 존중한다.
2. 요청 범위와 직접 관련된 부분만 최소 수정한다.
3. 확정되지 않은 정책/계약 변경은 임의로 단정하지 않는다.
4. 성공 경로뿐 아니라 실패/예외/상태 처리까지 함께 확인한다.
5. 점검 결과는 즉시 대응 항목과 배포 후 모니터링 항목으로 분리해 보고한다.

## 점검 절차

아래 1~6단계를 순서대로 수행한다.

### 1) 핵심 기능 정상 동작 확인

- 사용자 가치가 큰 핵심 시나리오 3~5개를 우선 점검한다.
- 기본 상태, 로딩 상태, 빈 상태, 오류 상태, 권한/제한 상태가 의도대로 동작하는지 확인한다.
- 기능 간 연결 흐름(예: 목록→상세→수정→저장)이 끊기지 않는지 확인한다.

### 2) 환경 설정/필수 변수 누락 여부 확인

- 실행 환경별 필수 변수 목록을 기준으로 누락/오타를 점검한다.
- PRD **측정=예**이면 analytics 키/host(또는 ingest URL) 누락·staging/prod 혼선을 점검한다.
- 배포 환경에서 사용하는 외부 연동 키/엔드포인트/리전 값이 맞는지 확인한다.
- 민감 정보가 코드/로그/문서에 노출되지 않는지 확인한다.

### 3) 모바일/웹 분기, 반응형, 다크모드 영향 확인

- 모바일/웹 분기 로직이 의도한 플랫폼에서만 적용되는지 확인한다.
- 모바일 앱이면 `docs/mobile/app-update/policy-and-contract.md` 릴리스 3항(minSupported 배포 일치, 구/권장/강제, API down)을 확인한다.
- 주요 브레이크포인트에서 레이아웃 깨짐, 잘림, 오버플로우를 점검한다.
- 다크모드에서 대비, 가독성, 상태 색상(성공/경고/오류) 일관성을 확인한다.

### 4) 콘솔 오류/네트워크 오류/예외 처리 누락 점검

- 브라우저 콘솔 또는 앱 로그에 오류/경고가 남는지 확인한다.
- 네트워크 실패(타임아웃, 4xx, 5xx, 오프라인) 시 사용자 안내와 재시도 동작을 점검한다.
- 예외 발생 시 화면이 중단되지 않고 복구 가능한 경로를 제공하는지 확인한다.
- PRD **측정=예**이면 `docs/product-analytics/release-checklist.md` **릴리스 3항**(env·동의·PII·샘플 이벤트)을 확인한다.
- PRD **성능 게이트=예**이면 `docs/performance/release-checklist.md` **릴리스 3항**(`perf-last.json`·활성 플랫폼·회귀)을 확인한다.

### 5) 필요한 문서/변경 요약 준비 여부 확인

- 변경 요약(무엇이, 왜 바뀌었는지)과 영향 범위를 최신화한다.
- 운영/QA/지원팀이 확인할 릴리즈 노트, 체크리스트, 롤백 가이드를 준비한다.
- 계약/API/정책 변경이 있으면 영향받는 범위를 명시한다.

### 6) 필요 시 qa-agent와 docs-agent 사용

- 검증 범위가 넓거나 회귀 위험이 크면 `qa-agent`를 호출해 교차 점검한다.
- 문서 정리, 릴리즈 노트, 인수인계가 필요하면 `docs-agent`를 호출한다.
- 병렬 점검 후에는 단일 통합 담당자가 최종 결과를 합쳐 보고한다.

## 결과물 형식

점검 완료 후 아래 형식으로 보고한다.

```markdown
판단 요약:
- 배포 가능 여부: [가능/조건부 가능/불가]
- 핵심 근거: [2~3줄]

실행 항목:
- [확인한 기능/환경/오류 처리 항목]

발견된 위험 요소:
- [위험 내용] / 영향도: [높음|중간|낮음] / 대응 상태: [미조치|진행중|완료]

즉시 수정 항목:
- [배포 전 반드시 수정할 항목]

배포 후 모니터링 포인트:
- [로그 지표, 에러율, 성능 지표, 사용자 제보 채널 등]
```

## 빠른 체크리스트

- [ ] 핵심 기능 정상 동작 확인 완료
- [ ] 환경 설정/필수 변수 누락 없음
- [ ] 모바일/웹 분기 및 반응형/다크모드 영향 확인 완료
- [ ] (모바일 앱) app-update 릴리스 3항 확인 완료
- [ ] (측정=예) product-analytics 릴리스 3항 확인 완료
- [ ] (성능 게이트=예) performance 릴리스 3항 확인 완료
- [ ] 콘솔/네트워크/예외 처리 누락 없음
- [ ] 문서/변경 요약 준비 완료
- [ ] 필요 시 qa-agent/docs-agent 협업 완료
- [ ] 배포 전 체크 결과와 위험/즉시 수정/모니터링 포인트 보고 완료
