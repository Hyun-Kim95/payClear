# payClear 용어집

PRD §4.1 기준. 코드·UI 표기는 한국어 사용자-facing / 영문 API enum 병기.

| 용어 | API·코드 | 설명 |
|------|----------|------|
| 기록자 | — | 로그인해 채무를 편집하는 본인 |
| 조회자(상대) | — | 공유 링크로 읽기만 하는 사람 |
| 상대 | Contact | 채무의 상대방 이름 단위(전화번호 필드는 **v0.2** 검토) |
| 채무 | Debt | 빌려줌·빌림 **한 건** |
| 빌려줌 | `lent` | 내가 줌 → 상대가 갚음 → **받을 금액** |
| 빌림 | `borrowed` | 내가 받음 → 내가 갚음 → **갚을 금액** |
| 원금 | `principal` | 생성 시 약정액, **수정 불가** |
| 상환 | `payment` | 갚은 기록(ledger), 금액 양수 |
| 조정 | `adjustment` | 합의·착오 반영(±), 사유 필수 |
| 잔액 | `balance` | 계산값, 직접 입력 없음 |
| 완료 | — | 잔액 0 + 자동 종료 라벨 |
| 합의 종료 | `agreement_closed` | 수동 종료. **뱃지**는 `status=completed`일 때만(P5b) |
| 표시 라벨 | `display_label` | `완료` \| `합의 종료` \| null |
| 연체 | `overdue` | active + 예정일 지남 + 잔액>0 |
| 보관 | `archived` | 목록 숨김, 편집 제한 |
| 공유 링크 | Share | 채무 1건 읽기 전용 URL |
| 초과 상환 | — | 잔액 < 0 |
| PWA | — | 홈 화면 추가·웹 푸시 가능한 웹 앱 |
