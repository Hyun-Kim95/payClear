## 2. 문제 정의·목표

### 2.1 현재 Pain

- 메모장에 **빌려준 금액 / 빌린 금액**, **이유·일자·상환 예정**, **부분 상환 내역**이 흩어져 있음.
- 시간이 지나면 “**지금 남은 금액이 왜 이 금액인지**” 설명이 어렵고, 상대와 말이 엇갈림.
- **누가·언제·얼마·왜·얼마 남았는지** 한 화면·한 흐름으로 보기 어려움.

### 2.2 제품 목표 (North Star에 가까운 정의)

> **임의 시점에 “누구와 어떤 채무 관계에서, 원금 대비 얼마를 갚았고, 왜 잔액이 지금 이 금액인지”를 30초 안에 말·화면으로 증명할 수 있다.**

### 2.3 성공 지표 (v0.1 — 정성, 추후 측정=예 시 계약화)

- 새 채무 1건 등록까지 **60초 이내** (필수 필드만).
- 상환 1건 기록 후 **잔액이 자동 반영**되고, 이력에서 재계산 가능.
- “X에게 아직 Y원” 질문에 **상세 화면 1곳**에서 답 가능.

### 2.4 비목표 (Out of Scope — v0.1)

- 실제 송금·계좌 연동·PG 결제
- 법적 차용증·전자서명·소송 지원
- 회계·세무·사업자 매출 신고
- 복수 사용자 협업 편집(가족 공동 장부 등)
- 신용평가·대출 중개

## 4. 도메인 모델 (개념)

### 4.1 용어

| 용어 | 정의 |
|------|------|
| **상대(Contact)** | 채무 관계의 상대 이름·메모 (전화번호 등 선택) |
| **채무(Debt)** | 한 상대와의 **한 건** 빌려줌 또는 빌림 계약 단위 |
| **방향** | `lent`(내가 빌려줌 → 상대가 갚음) / `borrowed`(내가 빌림 → 내가 갚음) |
| **원금(Principal)** | 채무 생성 시 최초 약정 금액(**불변**). 이후 금액 변경은 **조정**만(§7.1 P3). 잔액은 상환·조정으로만 변동(§4.2) |
| **상환(Payment)** | 일자·금액·메모가 있는 갚음 기록 (부분 상환 여러 건 가능) |
| **잔액(Balance)** | `원금 + 조정합 − 상환합` (방향에 따라 “받을 돈” / “갚을 돈” 라벨만 다름) |
| **조정(Adjustment)** | 원금 착오·합의 변경 등 **이유 필수** 이벤트 (**v0.1 포함·확정**) |
| **공유 링크(Share)** | 채무 1건에 대한 읽기 전용 URL. 만료·비밀번호(선택) 정책은 §7.4 |

### 4.2 잔액 계산 규칙 (정책 초안)

```
잔액 = principal_amount + sum(adjustments) - sum(payments)
```

- `lent`: 잔액 > 0 → “받을 금액”
- `borrowed`: 잔액 > 0 → “갚을 금액”
- 잔액 = 0 → **자동 완료 후보**(P5①, `agreement_closed` 없을 때). 이미 **합의 종료**면 라벨 유지(P5a)
- 잔액 < 0 → **초과 상환** (P6: 자동 완료 **안 함**)

### 4.3 채무 상태

> `status` 필드와 표시 라벨·`overdue` 뱃지는 별도. 잔액은 §4.2식으로 **항상 계산**.

| 상태 | 조건 |
|------|------|
| `active` | `status=active`(진행 중). 잔액은 0·양수·음수 모두 가능 |
| `completed` | `status=completed`. 잔액 0(자동·P5①) 또는 **>0 수동 합의 종료**(P5②). 라벨: 「완료」 또는 「합의 종료」 |
| `overdue` | `status=active` **이고** `due_on`<오늘(KST) **이고** 잔액>0. `completed`·`archived`는 **연체 아님** |
| `archived` | 사용자가 목록에서 숨김. 편집·알림·연체 표시 **제외**(X7) |

## 6. 화면·상태 (Gate 1 화면 스펙)

### 6.1 정보 구조 (IA)

```
[로그인/가입]
    └── [홈 · 요약]
            ├── 받을 돈 합계 / 갚을 돈 합계
            └── 최근 활동 · 다가오는 예정일
    └── [채무 목록]
            ├── 필터·검색
            └── 채무 카드(상대, 방향, 잔액, 상태뱃지, 예정일)
    └── [채무 상세]
            ├── 헤더(상대, 방향, 잔액, 상태)
            ├── 타임라인(원금·조정·상환)
            └── [+ 상환] [+ 조정] [공유 링크] [편집·완료·보관]
    └── [채무 등록/수정]
    └── [상대 목록 · 상대 상세(해당 채무 모음)]

    └── [잠금 화면]  ← F15
    └── [설정]
            ├── 계정 · 알림 · 잠금 · 로그아웃
```

### 6.3 UX 원칙 (본 제품)

- **잔액은 항상 계산값**으로 표시(수동 “남은 금액” 필드 없음).
- 상세 타임라인이 **“왜 이 금액인가”의 단일 진실 공급원**.
- 빌려줌/빌림은 **색·라벨·아이콘**으로 즉시 구분(색맹: 라벨 필수).
- 금액 입력: 천 단위 구분. 상환·원금은 **양수만**. 조정은 **양수/음수**(§7.1 P10) 별도 UI.

### 7.2 개인정보·보안

- 채무 데이터 **편집(CRUD)** 은 본인 계정만.
- **F15 잠금**은 **기록자 앱(로그인 영역)** 만. **공유 보기 URL(F16)** 에는 기록자 PIN 요구 **하지 않음**(링크 PIN만, §7.4).
- 서비스 약관·개인정보 처리방침: 배포 전 문서화(**미확정** 법무 검토).
- **F15**: 앱 PIN 또는 WebAuthn/생체(지원 시). **백그라운드 5분** 후 잠금(설정에서 1/5/15분).
- 공유·알림에 쓰는 **상대 표시명**은 기록자가 입력한 정보이며, 기록자 책임(면책 문구는 약관 초안).

### 7.3 알림 (F11 필수)

- 대상: `due_on` 있는 **active** 채무만. **completed·archived** 제외.
- 시점: 예정일 **1일 전 09:00 KST**, **당일 09:00 KST**(설정에서 off 가능).
- **채널 우선순위 (v0.1)**: ① Web Push(권한 허용 시) ② **이메일**(계정 **인증된 이메일 필수**). Push 불가·거부 시 **이메일만**으로도 F11 충족.
- 소셜 로그인만 있는 경우: 최초 로그인 후 **이메일 등록·인증** 플로우 **필수**(알림·계정 복구).
- iOS Safari PWA Push 제한: **이메일이 기본 폴백**이며, 설정에 “푸시 미지원 안내” 표시.

### 7.4 공유 링크 (F16 필수)

- 범위: **채무 1건**. 포함: 방향, 현재 잔액, 타임라인(개설·조정·상환), 사유(기록자 선택 시 제외 가능 — 기본 **포함**).
- **상대 표시**: 기본 `display_name`. 생성 시 **「익명(상대)」** 토글(P14).
- **금지**: 편집, 다른 채무·다른 상대 정보, 기록자 계정 정보.
- **무효화**: 기록자 **회수**, **보관·완전 삭제**(P2), **만료(P11)**, **계정 삭제(P13)** → 404 + 안내 문구.
- **활성 링크 1개/채무**: 새 링크 생성 시 **기존 링크 자동 회수**(재공유는 새 URL).
- 선택: 링크 **4~6자리 PIN**. 5회 실패 시 **15분 잠금**.
- 공유 페이지: “payClear에서 관리 중” + **조회 전용·법적 효력 없음** 고지.

## 부록 C — PWA란? (질문 1)

**PWA(Progressive Web App)** 는 **웹사이트를 앱처럼 쓰게 만드는 방식**입니다.

| 일반 웹 | PWA |
|---------|-----|
| 브라우저 주소창에서만 접속 | **홈 화면에 아이콘 추가** 후, 거의 앱처럼 전체 화면 실행 |
| 매번 URL 입력·북마크 | 한 번 설치(「홈에 추가」)하면 아이콘 탭으로 실행 |
| 오프라인·알림 제한적 | **오프라인 일부 지원**, **푸시 알림**(기기·브라우저 허용 시) 가능 |

**payClear에 쓰면 좋은 점**

- 앱스토어 심사·별도 iOS/Android 개발 없이 **빠르게** “폰에 깔린 느낌” 제공
- 로그인+클라우드·공유 링크는 **웹 URL** 그대로 동작
- 상환 알림(F11)을 **푸시**로 보낼 수 있음(Android·데스크톱은 비교적 수월, **iPhone은 제한**이 있어 이메일 폴백을 PRD에 넣음)

**단점·주의**

- 네이티브 앱만큼 OS 깊은 연동(위젯, 전화번호부 자동 연동 등)은 어렵거나 늦춤
- iPhone 사용자가 많으면 **알림이 안 올 수 있음** → v0.1에서 이메일 리마인더 보조 권장

**PRD 추천 (U1)**  
v0.1 = **반응형 웹 + PWA 설치·푸시**. 나중에 필요하면 스토어 **네이티브 앱**을 v0.3에서 추가.

**확정 (2026-06-04)**  
v0.1 = **웹+PWA 먼저**. 스토어 앱은 v0.2 이후 필요 시.
